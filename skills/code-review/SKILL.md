# Code Review Skill

## 📋 定义

Code Review 检查清单，确保代码质量和最佳实践。

## 🎯 触发场景

- ✅ Pull Request 审查
- ✅ Merge Request 审查
- ✅ 代码提交前自查
- ✅ 团队代码互审

## 📏 检查清单

### 1. 代码质量 (30%)

- [ ] 代码通过所有测试
- [ ] 新增测试覆盖新功能
- [ ] 无 lint 错误
- [ ] 类型检查通过 (mypy)
- [ ] 代码格式化符合规范 (black, isort)
- [ ] 无重复代码 (DRY 原则)
- [ ] 函数/方法简洁单一职责

### 2. 功能正确性 (25%)

- [ ] 实现符合需求描述
- [ ] 边界条件处理完善
- [ ] 错误处理合理
- [ ] 输入验证完整
- [ ] 无逻辑错误
- [ ] 性能考虑充分

### 3. Harness Engineering (20%)

- [ ] Guardrails 覆盖率 100%
- [ ] Validators 覆盖率 100%
- [ ] Monitors 指标完整
- [ ] 异常自动修复机制
- [ ] 监控告警配置

### 4. 文档 (15%)

- [ ] README 已更新
- [ ] CHANGELOG 已更新
- [ ] API 文档同步
- [ ] 注释清晰准确
- [ ] Commit Message 规范

### 5. 安全 (10%)

- [ ] 无硬编码敏感信息
- [ ] 依赖项安全审查通过
- [ ] 无 SQL/XSS 注入风险
- [ ] 权限验证完善
- [ ] 日志不包含敏感数据

## 🔧 审查流程

1. **自动检查** - CI/CD 流水线
2. **功能审查** - 代码是否按预期工作
3. **质量审查** - 代码质量和规范
4. **安全审查** - 安全性检查
5. **批准合并** - 至少 1 人批准
